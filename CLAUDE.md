# CLAUDE.md — project rules

> npm scope note: `@hookkit` was already taken on npm, so this project uses the free scope
> `@hookkit-dev` for every package (`@hookkit-dev/core`, `@hookkit-dev/sdk`, …). Keep the
> scope consistent everywhere. The CLI binary is still named `hookkit`.

## Product invariants (never break these)
- OFFLINE-FIRST: `core`, `sdk`, `cli` must function with zero network access. Their test
  suites must pass with networking disabled. No package phones home. No telemetry, ever.
- NO ACCOUNT / NO CLOUD: no API keys, no sign-up, no HookKit-operated server anywhere in
  the default paths. The ONLY network component is the optional, USER-self-hosted
  `@hookkit-dev/relay`. Never route user traffic through a server we operate.
- RAW-BODY FIDELITY: the exact received/generated bytes (a Buffer) are the source of truth
  end to end. Never JSON.parse→JSON.stringify a payload before signing or delivery.
- PROVIDER-ACCURATE SIGNING: a generated event MUST verify against the provider's official
  verification library. This is enforced by golden tests; do not merge a provider without one.
- SIGNING USES `node:crypto` ONLY. No third-party crypto deps.
- LICENSE: MIT. Fixtures contain synthetic data only — no real PII, tokens, or secrets.

## Engineering rules
- TypeScript strict everywhere. ESM output (with CJS interop via tsup). Node >= 20.
- Provider logic lives ONLY in @hookkit-dev/core. Surfaces import it.
- Secrets are read from env or flags, never logged, never written to disk.
- Every provider adapter must pass the shared conformance suite before it's considered done.

## Verify before done
- `pnpm verify` must be green (lint + typecheck + test + build).
